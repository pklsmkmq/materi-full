    State Management Provider untuk List Data (CRUD Dasar)

    Tujuan Pembelajaran

    Setelah mengikuti pembelajaran ini, siswa mampu:
    1. Mengelola state berbentuk list menggunakan Provider
    2. Melakukan operasi tambah dan hapus data
    3. Memisahkan data, logic, dan UI
    4. Menyiapkan pondasi sebelum API

    Pertanyaan Pemantik

    Bayangkan aplikasi daftar tugas yang datanya terus bertambah.
    Apa yang terjadi jika data disimpan di UI masing-masing halaman?

    Materi Inti

    State berbentuk list sering digunakan di aplikasi nyata:
    - daftar produk
    - daftar tugas
    - daftar transaksi

    List ini harus dikelola terpusat agar konsisten.

    Tutorial Praktik

    Membuat Model

    File: lib/models/task.dart

    class Task {
  String title;
  bool isDone;

  Task({required this.title, this.isDone = false});
}

    Membuat Provider

    File: lib/providers/task_provider.dart

    import 'package:flutter/material.dart';
import '../models/task.dart';

class TaskProvider extends ChangeNotifier {
  final List<Task> _tasks = [];

  List<Task> get tasks => _tasks;

  void addTask(String title) {
    _tasks.add(Task(title: title));
    notifyListeners();
  }

  void removeTask(int index) {
    _tasks.removeAt(index);
    notifyListeners();
  }
}

    Menggunakan Provider di UI

    Contoh di halaman TaskPage.

    final taskProvider = context.watch<TaskProvider>();

    Tugas

    Tugas Dasar

    • Tambahkan 5 data awal
    • Tampilkan jumlah task

    Tugas Industri

    • Tambahkan fitur edit task
    • Jangan simpan logic edit di UI
