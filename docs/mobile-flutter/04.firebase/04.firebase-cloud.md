## 1. Setup Cloud Firestore & Security Rules

Langkah pertama adalah membuat database NoSQL kita di Firebase Console.

**Langkah-langkah Setup:**

1. Buka dashboard project Firebase kalian.

2. Di menu sebelah kiri, pilih **Build** -> **Firestore Database**.

3. Klik tombol **Create database**.

4. Lokasi server: Pilih lokasi terdekat (misalnya `asia-southeast2` untuk Jakarta) agar aplikasi berjalan cepat. Lalu klik **Next**.

5. Pilih **Start in production mode** (Mulai dalam mode produksi). Klik **Create**.

6. Setelah database jadi, buka tab **Rules** (Aturan). 

7. Ubah aturan keamanannya menjadi seperti di bawah ini agar hanya *user* yang sudah login yang bisa membaca dan menulis data:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Kunci keamanan: Hanya user yang sudah login (!= null) yang bisa akses
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

8. Klik **Publish**. Database kalian sekarang aman!

## 3. Membuat Logika CRUD di Provider

Kita akan membuat satu file *Provider* khusus untuk menangani proses keluar-masuk data dari database. 

Buat file baru bernama `lib/providers/attendance_provider.dart`, lalu masukkan kode berikut:

```dart
import 'package:cloud_firestore/cloud_firestore.dart';
import 'package:flutter/material.dart';

class AttendanceProvider with ChangeNotifier {
  // Inisialisasi penghubung ke Firestore
  final FirebaseFirestore _firestore = FirebaseFirestore.instance;

  // 1. Fungsi CREATE: Menambah data logbook absensi ke Firestore
  Future<void> addAttendance(String uid, String status, String note) async {
    try {
      // Kita menembak ke "Collection" bernama 'attendances'
      await _firestore.collection('attendances').add({
        'student_id': uid,       // ID milik siswa (dari FirebaseAuth)
        'date': Timestamp.now(), // Waktu absen otomatis dari server
        'status': status,        // Hadir/Sakit/Izin
        'activity_note': note,   // Catatan kegiatan hari ini
      });
      notifyListeners();
    } catch (e) {
      throw Exception('Gagal menyimpan data: $e');
    }
  }

  // 2. Fungsi READ: Mengambil data absen milik 1 siswa secara REAL-TIME
  // Perhatikan: Kita mereturn tipe "Stream", bukan "Future"
  Stream<QuerySnapshot> getStudentAttendances(String uid) {
    return _firestore
        .collection('attendances')
        .where('student_id', isEqualTo: uid) // Filter khusus data siswa login
        .orderBy('date', descending: true)   // Urutkan dari yang terbaru
        .snapshots();                        // Dengarkan setiap ada perubahan data
  }
}
```

**Konsep Penting untuk Siswa:**
Perhatikan fungsi Read di atas. Di SQL, kita harus melakukan *Refresh* (Query ulang) setiap kali ada data baru. Di Firestore, fungsi `.snapshots()` membuat aplikasi kita **terus memantau** database. Jika ada data baru, UI aplikasi akan otomatis terupdate tanpa perlu di-refresh!

## 4. Menampilkan Data di UI (Dashboard)

Untuk menampilkan aliran data *real-time* (Stream) dari Provider di atas, Flutter menyediakan *widget* khusus bernama `StreamBuilder`. 

Buka file `lib/screens/dashboard_screen.dart` dan gunakan kode seperti ini untuk menampilkan daftar logbook:

```dart
import 'package:cloud_firestore/cloud_firestore.dart';
import 'package:firebase_auth/firebase_auth.dart';
import 'package:flutter/material.dart';
import 'package:provider/provider.dart';
// import provider kalian...

// ... (di dalam method build UI Anda)

StreamBuilder<QuerySnapshot>(
  // 1. Panggil fungsi Stream dari Provider
  stream: context.read<AttendanceProvider>().getStudentAttendances(
    FirebaseAuth.instance.currentUser!.uid
  ),
  builder: (context, snapshot) {
    // 2. Tampilkan efek loading jika data sedang diambil
    if (snapshot.connectionState == ConnectionState.waiting) {
      return const Center(child: CircularProgressIndicator());
    }

    // 3. Tampilkan pesan jika data kosong / error
    if (!snapshot.hasData || snapshot.data!.docs.isEmpty) {
      return const Center(child: Text('Belum ada logbook/absensi.'));
    }

    // 4. Jika data ada, ekstrak menjadi ListView
    return ListView.builder(
      itemCount: snapshot.data!.docs.length,
      itemBuilder: (context, index) {
        // Mengubah dokumen NoSQL menjadi Map/Dictionary
        var data = snapshot.data!.docs[index].data() as Map<String, dynamic>;
        
        return Card(
          child: ListTile(
            title: Text(data['activity_note']), // Menampilkan catatan
            subtitle: Text('Status: ${data['status']}'), // Menampilkan status
          ),
        );
      },
    );
  },
)
```

Dengan langkah ini, siswa sudah berhasil mempraktekkan *Data Modeling* NoSQL (menyimpan `student_id` sebagai penghubung data) sekaligus menampilkan datanya ke dalam aplikasi secara *real-time*.
