# Pengenalan State Management Provider

---

## Pertanyaan Pemantik
Pernahkah kalian mengalami aplikasi yang:  
- datanya tidak ikut berubah walaupun sudah diupdate?  
- harus bolak-balik refresh layar?  
- kodenya makin lama makin sulit dipahami?

Menurut kalian, kenapa hal itu bisa terjadi?

---


## 1. Apa itu State?
State adalah **data yang bisa berubah** selama aplikasi berjalan.

Contoh state:  
- jumlah barang di keranjang  
- status login (sudah login atau belum)  
- daftar produk  
- data user

Jika state berubah, **tampilan aplikasi seharusnya ikut berubah**.

---

## 2. State dalam Kehidupan Sehari-hari
Bayangkan papan tulis di kelas:  
- Isi papan tulis = state  
- Guru mengubah tulisan = state berubah  
- Murid melihat papan tulis = UI

Jika tulisan berubah tapi murid tidak melihat perubahan,
berarti ada masalah dalam penyampaian state.

---

## 3. Mengelola State Tanpa State Management
Pada aplikasi kecil, kita sering:  
- menyimpan data di widget  
- mengoper data lewat constructor  
- mengandalkan setState

Cara ini masih aman untuk:  
- 1â€“2 halaman  
- data sederhana

Namun akan bermasalah jika:  
- data dipakai di banyak halaman  
- aplikasi semakin besar  
- banyak fitur saling terhubung

---

## Studi Kasus Awal
Bayangkan aplikasi kasir:  
- halaman produk  
- halaman keranjang  
- halaman checkout

Jika data keranjang disimpan di masing-masing halaman:  
- data bisa tidak sinkron  
- jumlah barang bisa berbeda  
- bug sulit dilacak

---

## Praktik: Contoh Masalah Tanpa State Management

### Kasus: Keranjang Sederhana
Kita akan mensimulasikan masalah umum.

Tambahkan kode berikut di salah satu halaman (contoh HomePage):

```dart
int cartCount = 0;
```

Lalu buat tombol:

```dart
ElevatedButton(
  onPressed: () {
    setState(() {
      cartCount++;
    });
  },
  child: Text('Tambah ke Keranjang: $cartCount'),
)
```

Masalah yang muncul:  
- data hanya hidup di halaman ini  
- halaman lain tidak tahu perubahan cartCount  
- jika aplikasi bertambah besar, kode makin sulit diatur

---

## Kenapa Ini Masalah di Dunia Kerja?
Di dunia kerja:  
- satu aplikasi bisa dikerjakan banyak orang  
- satu data dipakai di banyak halaman  
- bug kecil bisa berdampak besar

Tanpa state management:  
- kode sulit dirawat  
- perubahan kecil bisa merusak banyak bagian  
- developer baru butuh waktu lama memahami alur

---

## Kesimpulan Sementara
State management dibutuhkan untuk:  
- menyimpan data di satu tempat  
- memastikan semua halaman melihat data yang sama  
- membuat aplikasi stabil dan scalable

Namun:  
- kita **tidak langsung pakai Provider hari ini**  
- hari ini fokus memahami masalahnya terlebih dahulu

## Tugas

### Tugas Level Dasar  
1. Sebutkan minimal 5 contoh state dalam aplikasi mobile  
2. Jelaskan dengan kalimat sendiri apa itu state  
3. Sebutkan 3 masalah jika state tidak dikelola dengan baik

---

### Tugas Level Industri  
1. Bayangkan aplikasi absensi online  
2. Sebutkan state apa saja yang mungkin ada  
3. Jelaskan risiko jika setiap halaman menyimpan state masing-masing

Tuliskan jawaban secara terstruktur.
